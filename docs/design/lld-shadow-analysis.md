# Component: Shadow Replay Analysis CLI

## Owner

Platform Engineer

## Purpose

Add CLI commands to `seictl report` for analyzing shadow replay comparison data stored in S3. Engineers and agents invoke the same CLI — agents pass `--json` for structured output. No MCP, no new protocols, no new dependencies.

**Business need:** The comparison pipeline (`result_compare.go`) writes structured comparison data to S3, but the only way to consume it is manually downloading and parsing gzipped NDJSON. These commands provide range-wide summarization, search, and trend analysis.

## Dependencies

| System | Interface | Notes |
|--------|-----------|-------|
| S3 (`{env}-sei-shadow-results`) | AWS SDK v2 `ListObjectsV2`, `GetObject` | Read-only. Default credential chain (IRSA in-cluster, env/profile locally). |
| `sidecar/shadow/` | `CompareResult`, `DivergenceReport`, `RenderMarkdown()` | Types and rendering. No modifications. |

**Does NOT:** modify the comparison engine, the export pipeline, the sidecar task engine, or the OpenAPI spec. No new Go dependencies.

## S3 Data Format (source of truth)

Written by `result_compare.go`. Two object types:

**Comparison pages** — `{prefix}{start}-{end}.compare.ndjson.gz`
- 100 blocks per page (`comparePageSize` const in `result_compare.go:19`)
- Gzipped NDJSON, each line is a `shadow.CompareResult`
- Key is generated by: `fmt.Sprintf("%s%d-%d.compare.ndjson.gz", prefix, start, end)` (`result_compare.go:198`)

**Divergence reports** — `{prefix}divergence-{height}.report.json.gz`
- Single gzipped JSON `shadow.DivergenceReport`
- Key is generated by: `fmt.Sprintf("%sdivergence-%d.report.json.gz", prefix, height)` (`result_compare.go:163`)

**Key parsing regexes:**
```go
var comparePageRe = regexp.MustCompile(`(\d+)-(\d+)\.compare\.ndjson\.gz$`)
var divergenceReportRe = regexp.MustCompile(`divergence-(\d+)\.report\.json\.gz$`)
```

**Disambiguation from raw export pages:** Raw export pages use `{start}-{end}.ndjson.gz` (no `.compare.`). Filtering on `strings.HasSuffix(key, ".compare.ndjson.gz")` is sufficient. As a safety net, validate decoded `CompareResult` has non-empty `Timestamp` (raw exports use `exported_at` instead).

## S3 Interfaces

The analysis package needs two S3 operations: listing objects and downloading objects. Use the existing `ObjectLister` interface for listing. For downloading, use the raw `s3.Client.GetObject` (NOT the `TransferClient` transfer manager, which requires `io.WriterAt` and defeats streaming).

**New interface in `sidecar/s3/client.go`:**

```go
// Downloader abstracts S3 GetObject for streaming reads.
type Downloader interface {
    GetObject(ctx context.Context, input *s3.GetObjectInput, opts ...func(*s3.Options)) (*s3.GetObjectOutput, error)
}

// DownloaderFactory builds a Downloader for a given region.
type DownloaderFactory func(ctx context.Context, region string) (Downloader, error)

// DefaultDownloaderFactory creates a real S3 client.
func DefaultDownloaderFactory(ctx context.Context, region string) (Downloader, error) {
    cfg, err := config.LoadDefaultConfig(ctx, config.WithRegion(region))
    if err != nil {
        return nil, fmt.Errorf("loading AWS config: %w", err)
    }
    return s3.NewFromConfig(cfg), nil
}
```

This follows the existing pattern of `ObjectListerFactory`, `UploaderFactory`, etc. in `sidecar/s3/client.go`.

## Store

```go
// sidecar/shadow/analysis/store.go
package analysis

type Store struct {
    lister     seis3.ObjectLister
    downloader seis3.Downloader
    bucket     string
    prefix     string
}

func NewStore(lister seis3.ObjectLister, downloader seis3.Downloader, bucket, prefix string) *Store
```

All analysis functions are methods on `Store`. The CLI constructs a `Store` from resolved flags. Tests inject mock `ObjectLister` and `Downloader`.

**Shared helper (in `analysis/store.go`):**

```go
// ResolveRef converts --env/--bucket/--prefix/--region flags to concrete values.
func ResolveRef(env, bucket, prefix, region string) (resolvedBucket, resolvedPrefix, resolvedRegion string, err error) {
    switch {
    case env != "" && bucket != "":
        return "", "", "", fmt.Errorf("--env and --bucket are mutually exclusive")
    case env != "":
        resolvedBucket = env + "-sei-shadow-results"
    case bucket != "":
        resolvedBucket = bucket
    default:
        return "", "", "", fmt.Errorf("one of --env or --bucket is required")
    }
    resolvedPrefix = prefix
    if resolvedPrefix == "" {
        resolvedPrefix = "shadow-results/"
    }
    if resolvedPrefix[len(resolvedPrefix)-1] != '/' {
        resolvedPrefix += "/"
    }
    resolvedRegion = region
    if resolvedRegion == "" {
        resolvedRegion = "eu-central-1"
    }
    return
}
```

## PageIterator

```go
// sidecar/shadow/analysis/iterator.go

// PageIterator streams CompareResults from S3 comparison pages.
// It downloads pages sequentially, decompresses gzip, and decodes
// NDJSON line-by-line. Results outside [start, end] are skipped internally.
type PageIterator struct {
    ctx        context.Context
    downloader seis3.Downloader
    bucket     string
    keys       []string       // sorted by parsed start height
    keyIdx     int
    start      int64          // inclusive lower bound
    end        int64          // inclusive upper bound
    reader     io.ReadCloser  // current page's gzip reader
    decoder    *json.Decoder  // current page's NDJSON decoder
}

// Next returns the next CompareResult within [start, end].
// Returns io.EOF when all pages are exhausted.
func (p *PageIterator) Next() (*shadow.CompareResult, error)

// Close releases the current page reader.
func (p *PageIterator) Close() error
```

**Implementation details:**
- `Next()` calls `json.Decoder.Decode()` on the current page. When a page is exhausted, it closes the reader and opens the next page.
- Results with `Height < start` or `Height > end` are skipped silently.
- Validates `result.Timestamp != ""` to catch wrong NDJSON format.
- Pages are downloaded **sequentially** in v1. If profiling shows S3 latency dominates for large ranges, add a prefetch goroutine behind the same interface later.
- Pages are sorted by parsed start height (not lexicographic S3 key order).

**Opening an iterator (method on Store):**

```go
// OpenRange returns a PageIterator over comparison pages overlapping [start, end].
// Returns an iterator that yields zero results (not an error) if no pages overlap.
// maxPages caps how many S3 pages are downloaded; -1 means unlimited.
func (s *Store) OpenRange(ctx context.Context, start, end int64, maxPages int) (*PageIterator, error)
```

This method:
1. Calls `ListObjectsV2` with the store's bucket/prefix
2. Filters keys matching `comparePageRe`
3. Parses start/end heights from each key
4. Filters to pages where `pageEnd >= start && pageStart <= end`
5. Sorts by parsed start height
6. Truncates to `maxPages`
7. Returns a `PageIterator` over the matching keys

## Analysis Functions (methods on Store)

### `Store.List`

```go
func (s *Store) List(ctx context.Context, input ListInput) (*ListOutput, error)
```

Lists comparison pages and divergence reports via `ListObjectsV2`. Parses keys, filters to requested height range, returns inventory.

### `Store.Summarize`

```go
func (s *Store) Summarize(ctx context.Context, input SummarizeInput) (*SummarizeOutput, error)
```

Opens a `PageIterator` over `[start, end]`, streams results, accumulates:
- Total blocks, matched blocks, diverged blocks
- Per-L0-field mismatch counts (appHash, lastResultsHash, gasUsed)
- Per-L1-field mismatch counts (code, gasUsed, gasWanted, log, events, presence)
- List of divergent heights (capped at 1000)

### `Store.Search`

```go
func (s *Store) Search(ctx context.Context, input SearchInput) (*SearchOutput, error)
```

Opens a `PageIterator`, filters diverged results by optional `Layer` and `Fields` criteria, applies `Offset` and `Limit` pagination.

**Field matching:** When `--field gasUsed` is specified without `--layer`, it matches gasUsed divergences at either L0 or L1.

### `Store.Trends`

```go
func (s *Store) Trends(ctx context.Context, input TrendsInput) (*TrendsOutput, error)
```

Opens a `PageIterator`, assigns each result to a bucket (window) by `height / windowSize`, computes per-window divergence rates. Identifies:
- **Clusters:** contiguous runs of 2+ divergent blocks.
- **Transitions:** adjacent windows where divergence rate changes by >25 percentage points.

## CLI Commands

### Shared S3 flags

Defined once, reused by all subcommands. Use `cli.MutuallyExclusiveFlags` for `--env` / `--bucket`:

```go
var s3Flags = cli.MutuallyExclusiveFlags{
    {
        Flags: []cli.Flag{
            &cli.StringFlag{
                Name:  "env",
                Usage: "Environment shorthand (expands to '{env}-sei-shadow-results')",
            },
            &cli.StringFlag{
                Name:    "bucket",
                Sources: cli.EnvVars("SEI_RESULT_EXPORT_BUCKET"),
                Usage:   "S3 bucket name",
            },
        },
    },
}

// Additional shared flags (not mutually exclusive):
// --prefix  string  (env: SEI_RESULT_EXPORT_PREFIX, default: "shadow-results/")
// --region  string  (env: SEI_RESULT_EXPORT_REGION, default: "eu-central-1")
// --json    bool    (default: false)
```

### Command definitions

**`seictl report list`**

```
Flags: [s3Flags] --start int64  --end int64  --limit int(100)
Action: runReportList
```

Human output: table with columns `TYPE | HEIGHT RANGE | SIZE`.
JSON output: `ListOutput` struct.
Exit 0 even if no data found (empty table).

**`seictl report summary`**

```
Flags: [s3Flags] --start int64(required)  --end int64(required)  --max-pages int(100)
Action: runReportSummary
```

Human output: key-value block:
```
Range:            198000000 - 198100000 (100000 blocks, 1000 pages)
Match rate:       99.7% (99700 / 100000)
Diverged blocks:  300

Layer 0 breakdown:
  AppHash:           280
  LastResultsHash:   15
  GasUsed:           5

Layer 1 breakdown:
  code:              3
  gasUsed:           12
  events:            8
```

JSON output: `SummarizeOutput` struct.
Exit 0 even if no data found (zeros).

**`seictl report search`**

```
Flags: [s3Flags] --start int64  --end int64  --layer int  --field string(repeatable)  --limit int(50)  --offset int(0)
Action: runReportSearch
```

Valid `--field` values:
- L0: `appHash`, `lastResultsHash`, `gasUsed`
- L1: `code`, `gasUsed`, `gasWanted`, `log`, `events`, `presence`

Human output: table with columns `HEIGHT | LAYER | FIELDS`.
JSON output: `SearchOutput` struct.

**`seictl report trends`**

```
Flags: [s3Flags] --start int64(required)  --end int64(required)  --window-size int64(10000)
Action: runReportTrends
```

Flag is `--window-size` (not `--bucket-size` to avoid confusion with `--bucket`).

Human output: table of windows + cluster/transition annotations.
JSON output: `TrendsOutput` struct.

**`seictl report divergence` (extended)**

Existing flags `--bucket --key --region --json` remain but are no longer required. New flags:

```
--env string      Environment shorthand
--height int64    Block height (computes key automatically)
```

`--key` and `--height` are mutually exclusive. When `--height` is provided, the key is computed as `{prefix}divergence-{height}.report.json.gz`.

Human output: existing `shadow.RenderMarkdown()`.
JSON output: existing `shadow.DivergenceReport` (no flattening — the `any`-typed fields are fine for CLI JSON).

## Deferred (Do Not Build)

| Feature | Rationale |
|---------|-----------|
| MCP server | CLI with `--json` is sufficient. Add MCP when a concrete agent integration requires it. |
| Concurrent page downloads | Sequential is correct for v1. Profile before adding prefetch complexity. |
| Human-readable output polish | v1 uses simple `fmt.Fprintf` tables. Iterate on formatting based on usage. |
| Live `compare` subcommand | The existing `shadow.Comparator` works; a CLI wrapper can be added separately. |
| Layer 2/3 analysis | Data doesn't exist yet. |
| Caching | S3 reads are fast enough at expected query volume. |
