# Cluster CLI: Low-Level Design

**Status:** Draft (design)
**Scope:** New cluster-facing subcommands for `seictl` — `bench`, `onboard`, `status`, `seinode`, `controller`, `context`
**Authors:** sei-platform-engineer skill council (platform-engineer lead, kubernetes-specialist, security-specialist)
**Last updated:** 2026-04-26
**Tracking:** sei-protocol/Tide#8 (sei-platform-engineer skill draft, depends on this)

## Summary

`seictl` is gaining a family of cluster-facing commands that turn it from a node-operator local tool into the engineer's interface for provisioning testing environments on the harbor EKS cluster. v1 ships:

| Verb | Purpose |
|---|---|
| `seictl bench up\|down\|list` | Render and apply benchmark workloads (validators + RPC + seiload Job) |
| `seictl onboard` | Generate engineer namespace files + Terraform diff; open PR via `gh` |
| `seictl status` | What the engineer owns across the cluster |
| `seictl seinode list\|diagnose` | SeiNode CR list + phase-aware diagnostic report |
| `seictl controller inspect` | sei-k8s-controller health + recent reconcile errors |
| `seictl context` | Cluster + identity + RBAC ground truth |

The conversational layer over this CLI is the `sei-platform-engineer` skill at sei-protocol/Tide#8. This LLD is the implementation contract that skill depends on.

## Goals

- Engineer types one command (eventually one natural-language sentence to a Claude skill); benchmark runs.
- Subcommand JSON output is the **MCP tool contract** — schemas defined here are one-way doors after v1.
- Auth and IAM scoping leverages existing harbor patterns (Pod Identity, EKS access entries) — no new auth surface.
- Templates derive from autobake's source of truth in `sei-protocol/platform`; no fork.
- kubectl-plugin compatible from day 1 (`kubectl-sei` symlink works against the same binary).

## Non-goals

- Cells-design infrastructure (separate workstream, ~1–2 weeks out — see sei-protocol/Tide#8 references/interim-namespace-strategy.md)
- MCP server (a v2 wrap around the CLI, after the schemas in this LLD bake)
- Multi-cluster benchmark orchestration (harbor-only in v1; `--context` flag wired but not exercised)
- Auto-cleanup of expired benchmarks (engineer triggers `bench down` or revert PR)

## Dependencies

- New Go module deps:
  - `github.com/sei-protocol/sei-k8s-controller/api/v1alpha1` (typed CR access)
  - `sigs.k8s.io/controller-runtime/pkg/client`
  - `k8s.io/cli-runtime/pkg/genericclioptions` (kubeconfig + kubectl-plugin idiomatic)
  - `k8s.io/client-go/kubernetes` (likely already transitive — direct use for Pods/log, SelfSubjectRulesReview, Events)
  - `github.com/google/go-containerregistry/pkg/name` (image ref parsing)
  - AWS SDK v2 ECR client
- External engineer-side dependencies: `gh` CLI authenticated, kubeconfig with `harbor` context.
- Upstream coordination:
  - `sei-protocol/platform` `autobake/templates/*` (vendored into seictl `templates/` with CI drift check)
  - New TF file `terraform/aws/189176372795/eu-central-1/harbor/personal-cells.tf` (generated by `seictl onboard`, applied by platform team)
  - sei-protocol/Tide#8 (skill draft references this LLD's command surface)

## Architecture

### Package layout

```

    identity/                [new — engineer.json read/write]
    validate/                [new — input validation regexes (single source of truth)]
    redact/                  [new — error redaction middleware]
    clioutput/               [new — Envelope, ErrorBody, exit-code mapping]
  templates/                 [new embed.FS root]
    embed.go                 [//go:embed * → fs.FS]

    snd-validator.yaml.tmpl
    snd-rpc.yaml.tmpl

    bench-seiload-sa.yaml.tmpl
    kustomization.yaml.tmpl
  scripts/
    check-templates-drift.sh [new — diffs vendored templates against pinned upstream]
  Makefile                   [edit: test-templates, check-templates-drift, lint-strict, build-kubectl-plugin]
  docs/design/cluster-cli.md [this file]
```

**Why flat at repo root for top-level command files:** seictl's existing convention is one file per top-level verb (`config.go`, `genesis.go`, `serve.go`, `await.go`). Putting `bench.go`, `onboard.go`, etc. at root preserves the convention. Internal packages handle the implementation depth.

### Common flags

Defined once in `internal/clioutput`, inherited by every cluster-facing command. `genericclioptions.ConfigFlags` wires `--kubeconfig`, `--context`, and `--namespace` once at the root command — per-subcommand registration would conflict.

```go
&cli.StringFlag{Name: "format", Value: "json", Usage: "Output format: json|text", Validator: oneOf("json","text")}
&cli.StringFlag{Name: "kubeconfig", Sources: cli.EnvVars("KUBECONFIG")}
&cli.StringFlag{Name: "context", Usage: "kubeconfig context override"}
&cli.StringFlag{Name: "namespace", Aliases: []string{"n"}}
```

### Cross-cutting envelope

Every command's stdout output (success and failure) is wrapped in:

```go
type Envelope struct {
    Kind    string          `json:"kind"`              // e.g. "bench.up.result"
    Version string          `json:"version"`           // schema version, "v1"
    Data    json.RawMessage `json:"data,omitempty"`
    Error   *ErrorBody      `json:"error,omitempty"`
}
```

`ErrorBody` shape is in [Error model](#error-model) below.

### `seictl bench up`

```
seictl bench up --image <ref> --slug <slug>
                [--size s|m|l] [--duration <duration>]
                [--profile <path>] [--apply]
                [--templates-dir <path>] [--skip-headroom-check]
                [--ecr-account 189176372795] [--ecr-region us-east-2]
```

Required: `--image`, `--slug`. Defaults: size `s`, duration `30m`, profile = embedded `autobake_evm_transfer.json`, namespace = `eng-<alias>` from identity.

**Default behavior is dry-run.** `--apply` performs the server-side apply.

```go
type BenchUpResult struct {
    ChainID            string        `json:"chainId"`         // "bench-<alias>-<slug>"
    Slug               string        `json:"slug"`
    Namespace          string        `json:"namespace"`
    ImageRef           string        `json:"imageRef"`        // input ref (tag or digest)
    ImageDigest        string        `json:"imageDigest"`     // resolved sha256:...
    Size               string        `json:"size"`
    Validators         int           `json:"validators"`
    RPCNodes           int           `json:"rpcNodes"`
    Duration           string        `json:"duration"`        // Go duration string
    ResultsS3URI       string        `json:"resultsS3Uri"`
    DryRun             bool          `json:"dryRun"`
    Manifests          []ManifestRef `json:"manifests"`
    AppliedAt          *time.Time    `json:"appliedAt,omitempty"`
    TemplatesRevision  string        `json:"templatesRevision"` // SHA from templates/UPSTREAM
}
type ManifestRef struct{ Kind, Name, Namespace, Action string } // action: "create"|"update"|"unchanged"
```

Fleet sizes:

| Size | Validators | RPC | Default seiload workers |
|---|---|---|---|
| `s` | 4 | 1 | matches autobake `s` |
| `m` | 10 | 2 | per autobake `m` |
| `l` | 21 | 4 | per autobake `l` |

Chain ID convention: `bench-<alias>-<slug>` (engineer benchmarks); RPC SND is `bench-<alias>-<slug>-rpc`. Distinguishes from autobake's nightly `autobake-<run-id>`.

S3 results URI: `s3://harbor-sei-autobake-results/<chain-id>/<image-sha-12>/<chain-id>/report.log`. Same bucket as nightly autobake; chain-ID prefix partitions.

### `seictl seinode diagnose <name>` *(load-bearing for skill UX)*

```go
type DiagnoseResult struct {
    Name           string         `json:"name"`
    Namespace      string         `json:"namespace"`
    Phase          string         `json:"phase"`           // SeiNodePhase enum value
    PhaseAge       string         `json:"phaseAge"`        // Go duration string
    Conditions     []Condition    `json:"conditions"`

    Recommendation Recommendation `json:"recommendation"`
    RecentEvents   []EventLine    `json:"recentEvents"`
    InitLogs       map[string]string `json:"initLogs,omitempty"` // container -> last N lines (with --include-logs)
}
type Condition struct {
    Type, Status, Reason, Message string
    LastTransitionAt              time.Time
}
type FailedTask struct {
    Name        string    `json:"name"`         // e.g. "configure-genesis"
    Type        string    `json:"type"`         // task type from controller's PlannedTask
    Attempts    int       `json:"attempts"`
    LastError   string    `json:"lastError"`
    LastAttempt time.Time `json:"lastAttempt"`
}
type Recommendation struct {
    Category     string   `json:"category"`     // enum (see below) — ONE-WAY DOOR
    Severity     string   `json:"severity"`     // "info"|"warn"|"error"|"blocked"
    Summary      string   `json:"summary"`      // one sentence in plain English
    Details      string   `json:"details"`      // multi-line; markdown ok
    NextCommands []string `json:"nextCommands"` // copy-pasteable shell commands, ordered by likely usefulness
    DocsURL      string   `json:"docsUrl,omitempty"`
}
type EventLine struct {
    Type, Reason, Message string
    Count                 int
    LastSeen              time.Time
}
```

`Recommendation.Category` enum (initial set; adding new categories is non-breaking, renaming or removing is breaking):

- `snapshot-403` — Pod Identity association missing or wrong policy
- `genesis-fetch` — Genesis URL unreachable or ConfigMap not mounted
- `peer-discovery` — EC2 tag query returned 0 peers
- `finalizer-stuck` — `sei.io/seinode-finalizer` blocking deletion
- `httproute-misconfig` — Gateway parentRefs or AuthorizationPolicy denying
- `pod-identity` — Pod Identity association missing or token unreadable
- `failed-terminal` — `.status.phase=Failed`; controller stopped reconciling
- `unknown` — diagnosis insufficient

## Kubernetes integration

### Client choice

**Combined approach:**

- `sigs.k8s.io/controller-runtime/pkg/client` — typed CRUD on SeiNode, SeiNodeDeployment, and standard objects. Compile-time safety on `.Status.Phase`, `.Status.Plan.FailedTaskDetail`, etc. Same library the controller itself uses.
- `k8s.io/cli-runtime/pkg/genericclioptions` — kubeconfig + `--kubeconfig`/`--context`/`--namespace`/`-n` flag wiring with idiomatic precedence (env > flag > config). Required for kubectl-plugin compatibility (`kubectl-sei` symlink works without rewiring).
- `k8s.io/client-go/kubernetes` raw — Pod log streaming, `SelfSubjectRulesReview`, Events. controller-runtime's typed client lacks these subresource accesses.

Wrap behind `internal/kube`:

```go
type Client struct {
    CR         client.Client            // typed CRUD
    Core       kubernetes.Interface     // logs, SelfSubjectRulesReview, Events
    Discovery  discovery.DiscoveryInterface
    RESTConfig *rest.Config
    Namespace  string
    Context    string
}
func New(flags *genericclioptions.ConfigFlags) (*Client, error)
```

Every command takes `*kube.Client`. No command pokes raw clientsets.

### CR access without vendoring controller code

Add seictl as a Go module dep on `github.com/sei-protocol/sei-k8s-controller/api/v1alpha1` (types-only, zero controller logic). Register the scheme, then `c.Get(ctx, key, &v1alpha1.SeiNode{})` works with full IDE help.

`Unstructured` is rejected: `seinode diagnose` switches on `.status.phase`, walks `.status.plan.tasks`, matches `.status.conditions[type=Ready]`. Typed access keeps the diagnostic logic readable; unstructured forces fragile string-keyed `NestedString` calls and loses enum constants.

Constraint: `go.mod` pins a controller version. Treat `api/v1alpha1` as a published interface — bump deliberately. Consider this when sei-k8s-controller releases.

### Apply strategy

**Server-side apply (SSA)** with field manager `seictl-bench` (or `seictl-onboard` for the rare onboard apply). Field manager string is a one-way door — changing it abandons ownership of every existing object.

```go
patch := client.Apply
opts := []client.PatchOption{client.FieldOwner("seictl-bench"), client.ForceOwnership}
for _, obj := range rendered { kc.CR.Patch(ctx, obj, patch, opts...) }
```

Why SSA:

- Conflict detection on duplicate slugs across engineers (409 with the conflicting field manager listed).
- Idempotent re-runs of `bench up --slug foo` — re-applying the same manifest is a no-op, not Create-conflict scramble.
- Field-manager segregation: controller writes `status.*` with its own field manager (`sei-k8s-controller`); SSA naturally segregates so seictl never fights status writes.

`--apply` semantics:

- Default: render templates → client-side validation against discovery cache → print summary → stop.
- `--apply`: SSA patches.
- `--dry-run=server` (engineer flag): SSA with `client.DryRunAll` — round-trips through the API server's admission chain, surfaces CRD CEL validation without persisting. This is the right pre-flight for catching schema violations the controller would reject at reconcile time.

### `seinode diagnose` flow

```go
func Diagnose(ctx context.Context, kc *kube.Client, name, ns string) (*DiagnoseResult, error) {
    var sn v1alpha1.SeiNode
    if err := kc.CR.Get(ctx, client.ObjectKey{Namespace: ns, Name: name}, &sn); err != nil { ... }

    d := &DiagnoseResult{Name: name, Namespace: ns, Phase: string(sn.Status.Phase), Conditions: convertConditions(sn.Status.Conditions)}

    switch sn.Status.Phase {
    case v1alpha1.PhasePending:
        d.Recommendation = recommendController(kc, ns)
    case v1alpha1.PhaseInitializing:
        if sn.Status.Plan != nil && sn.Status.Plan.FailedTaskDetail != nil {
            d.FailedTask = convertFailedTask(sn.Status.Plan.FailedTaskDetail)
            if includeLogs {
                d.InitLogs = fetchPodLogs(ctx, kc, ns, name+"-0", initContainersFor(d.FailedTask.Type), 200)
            }
            d.Recommendation = recommendForTask(d.FailedTask.Type)
        }
    case v1alpha1.PhaseFailed:
        d.Recommendation = recommendFailed(sn.Status.Conditions)
    case v1alpha1.PhaseRunning:
        d.Recommendation = recommendSteadyState(kc, ns, name)
    }
    d.RecentEvents = listEvents(ctx, kc, ns, name)
    return d, nil
}
```

Pod log access: `kc.Core.CoreV1().Pods(ns).GetLogs(podName, &corev1.PodLogOptions{Container: c, TailLines: ptr.To[int64](200)}).Stream(ctx)`. Never exec. Pod name follows StatefulSet convention `<seinode>-0` (single-replica per SeiNode).

Diagnose logic and the recommendation map live in `internal/diagnose`. Easier to unit-test against fake clients than threaded through the command layer.

### `context` flow — RBAC self-check

`kc.Core.AuthorizationV1().SelfSubjectRulesReviews().Create(ctx, &authzv1.SelfSubjectRulesReview{Spec: authzv1.SelfSubjectRulesReviewSpec{Namespace: kc.Namespace}}, metav1.CreateOptions{})`.

Filter the `ResourceRules` down to GVRs seictl cares about (seinodes, seinodedeployments, configmaps, jobs, pods, pods/log) and report verb-by-verb. Same mechanism as `kubectl auth can-i --list`.

AWS principal: shell out to `aws sts get-caller-identity --output json` (one call; SDK overkill). Parse `Account` + `Arn`.

Cluster name: parse from server URL (EKS pattern `https://<id>.<region>.eks.amazonaws.com`) cross-checked against the kubeconfig context name regex.

### Label discipline

Every resource seictl creates carries:

| Label | Value | Set by |
|---|---|---|
| `app.kubernetes.io/managed-by` | `seictl` | always |
| `app.kubernetes.io/part-of` | `seictl-bench` or `seictl-onboard` | command |
| `sei.io/engineer` | engineer alias from `~/.seictl/engineer.json` | always |
| `sei.io/bench-slug` | the `--slug` value | bench up |
| `sei.io/bench-run-id` | ULID from `bench up` invocation | bench up |
| `tide.sei.io/cell-type` | `personal` | onboard (cells-forward) |
| `tide.sei.io/owner` | engineer alias | onboard (cells-forward) |

Queries:

- `bench list` → `app.kubernetes.io/part-of=seictl-bench,sei.io/engineer=<alias>` on SeiNodeDeployments
- `bench down --slug X` → add `sei.io/bench-slug=X`. Get SNDs first, delete with foreground propagation so children cascade in order
- `status` → `sei.io/engineer=<alias>` across SeiNode, SeiNodeDeployment, ConfigMap, Job

Constraint: don't rely on the controller's auto-stamped `sei.io/nodedeployment` label for owner-scoped queries — it identifies parent SND, not the engineer.

### Landmine: `bench down` and PVC finalizers

`sei.io/seinode-finalizer` blocks SeiNode deletion until the controller releases PVCs cleanly. If the controller is unhealthy, the leader lease is held by a dead pod, or EBS CSI flakes on volume detach (common on harbor), `kubectl delete` returns immediately but children sit `Terminating` forever — eating quota and silently failing future `bench up --slug X` on duplicate-name conflicts.

`bench down --apply` **must**:

1. Issue delete with `metav1.DeletePropagationForeground` (server-side blocks until children are gone)
2. Wrap in `context.WithTimeout(90s)`
3. On timeout: don't auto-patch finalizers. Surface still-terminating resources, auto-run `controller inspect`, and print the explicit recovery command (`kubectl patch seinode <n> -p '{"metadata":{"finalizers":[]}}' --type=merge`). Engineer decides.

Per skill halt conditions: never auto-remediate. The `--force` flag is tempting; **don't add it in v1**.

## Security cross-cuts

### Input validation matrix

Single source of truth in `internal/validate/`. One function per input type returning typed sentinel errors. Commands call `validate.Alias(s)` etc. before any side effect.

| Input | Validation | Failure exit | Notes |
|---|---|---|---|
| `alias` | `^[a-z]([a-z0-9-]{0,28}[a-z0-9])?$` AND not in deny-list `{kube-system, kube-public, kube-node-lease, default, autobake, flux-system, istio-system, prod, mainnet, pacific, tide-agents}` | 20 | onboard; security-sensitive |
| `slug` | `^[a-z0-9]([a-z0-9-]{0,38}[a-z0-9])?$`. Combined `bench-<alias>-<slug>` ≤ 63 chars (k8s label limit) | 12 | bench up |
| `--image <ref>` | (a) Parse with `name.ParseReference`. (b) Hostname **must equal** `189176372795.dkr.ecr.us-east-2.amazonaws.com`. (c) Repo prefix **must be** `sei/`. (d) Resolve to digest before render; fail closed | 10 | security-critical — see [Security regressions](#security-regressions-if-shipped-wrong) |
| `--size` | Strict enum `s\|m\|l` | 12 | |
| `--duration` | Integer minutes, `1 ≤ n ≤ 240` | 12 | DoS-relevant: caps fleet pinning |
| Engineer IAM principal | Derived from `aws sts get-caller-identity` only. **No `--principal-arn` flag.** Cross-check ARN against `^arn:aws:iam::189176372795:role/sso-engineer-<alias>$` literal alias match | 12 | |
| `namespace` | RFC-1123 label `^[a-z0-9]([a-z0-9-]{0,61}[a-z0-9])?$` AND must equal `eng-<engineer.alias>` for any side-effecting verb | 13 | Read-only verbs may target other namespaces |

### IAM scoping for `onboard`

**[OPEN — see Open questions]** Two positions on the IAM policy that engineer cells should bind to:

**Position A (interim plan, in skill draft):** Reuse existing `harbor-autobake-seiload` IAM policy. Simplest. One TF change per engineer. **Drawback:** every engineer can write anywhere in `harbor-sei-autobake-results`, including overwriting nightly autobake reports.

**Position B (security recommendation):** Ship a new `harbor-bench-seiload-shared` policy scoped to `s3://harbor-sei-autobake-results/bench-*/` with explicit `Deny` on `arn:aws:s3:::harbor-sei-autobake-results/autobake-*/*`. ### Auth flow (pre-flight order, safety-first)

For any side-effecting command:

1. **Local input validation** — pure, no I/O. Catches typos before any AWS side effect lands in CloudTrail.
2. **Identity file** — `~/.seictl/engineer.json` exists, parses, perms `0600`.
3. **kubectl context** — current context matches `harbor` or `dev`, NOT prod. Production refusal here, before any AWS describe call (an engineer with prod kubeconfig + harbor AWS creds otherwise leaks intent into the wrong CloudTrail).
4. **AWS caller identity** — `aws sts get-caller-identity`. Account = `189176372795`, role pattern matches `sso-engineer-<alias>` or platform admin. Wrong account fails before any describe.
5. **Image registry policy + ECR digest** — whitelist hostname FIRST, then `ecr:DescribeImages`.
6. **Namespace existence + ownership** — `kubectl get ns eng-<alias>` AND `tide.sei.io/owner` label equals `<alias>`. Without label check, an engineer renamed in `engineer.json` could drive workloads into someone else's namespace.
7. **Pod Identity association sanity** (best-effort, non-blocking warning) — `aws eks describe-pod-identity-association` for `(eng-<alias>, bench-seiload)`; if absent, warn "your onboarding PR may not be merged yet."
8. **Render** manifests in-memory; show pre-flight echo. Engineer confirms.
9. **Apply.**

### Identity file: `~/.seictl/engineer.json`

- Mode `0600`, parent dir `0700`. On read, if perms `> 0600`: refuse and tell engineer to `chmod 0600`. (Skip on Windows or use `os.UserConfigDir` ACL semantics.)
- Three fields only: `alias`, `name`, `email`. Validate on read.
- Confidentiality is irrelevant; **integrity matters**: a loose-perms file on a shared workstation is a path to onboarding-as-someone-else or benching into their namespace.
- Pod Identity tokens are kubelet-managed; seictl never sees, mints, or proxies them.

### Security regressions if shipped wrong

Three items on the "ship-blocker" list:

1. **Image registry whitelist must be exact-match, applied before ECR resolution, and cover both `--image` AND any TF-rendered image refs.** A `*amazonaws.com` wildcard or contains-check lets `189176372795.dkr.ecr.us-east-2.amazonaws.com.attacker.tld` land. Pin the hostname literal; pin repo prefix `sei/`; resolve to digest; render the digest, never the tag. Manifests must always emit `image: <registry>/<repo>@sha256:<digest>`.
2. **Engineer IAM principal must come from `aws sts get-caller-identity` cross-checked against `engineer.json`, never from a free-form CLI flag.** No `--principal-arn` override. Otherwise an engineer authenticated as a break-glass admin can write a TF block wiring admin to someone else's Pod Identity.
3. **`~/.seictl/engineer.json` integrity (`0600` + parent `0700`) plus refuse-on-loose-perms.** Loose-perms file on a shared workstation is a confused-deputy path to onboarding-as-someone-else or benching into another engineer's namespace.

## Embedded templates strategy

`go:embed` a vendored copy under `templates/`, with a CI check that diffs against pinned upstream commit in `sei-protocol/platform`.

```
templates/
  embed.go                          //go:embed * → fs.FS
  UPSTREAM                          # plain text: "sei-protocol/platform@<sha> autobake/templates"
  snd-validator.yaml.tmpl
  snd-rpc.yaml.tmpl
  seiload-job.yaml.tmpl
  autobake_evm_transfer.json.tmpl
  namespace.yaml.tmpl               # for onboard
  bench-seiload-sa.yaml.tmpl        # for onboard
  kustomization.yaml.tmpl
scripts/check-templates-drift.sh   # diffs vendored against pinned upstream@SHA
```

Why vendoring over live-fetch:

- Reproducible builds — no network at build time, no broken goreleaser tag builds when upstream files move.
- Same discipline as `go.mod` pinning sei-chain.
- CI drift check is a nag, not a blocker; intentional divergence allowed by bumping `templates/UPSTREAM`.
- Engineer iteration: `--templates-dir <path>` overrides the embed.FS for local autobake checkouts.

Provenance: `seictl bench up` includes `templatesRevision` (the SHA from `templates/UPSTREAM`) in `BenchUpResult` JSON. Engineer's results carry a provenance trail.

### Rendering

`text/template` — autobake's source files use Helm-style `{{ }}` already. Rendered to YAML, parsed via `yaml.UnmarshalStrict` into typed `client.Object` slice, applied via SSA. No Kustomize at runtime in seictl (engineer's onboarding kustomization.yaml is for Flux to consume in the platform repo, not seictl-applied).

## Error model

When a multi-step command fails partway, the engineer sees a structured envelope with full state:

```go
// Already shown above. Repeated for emphasis on the "Completed/Remaining" recovery contract:
type ErrorBody struct {
    Code         int       // exit code
    Category     string    // stable enum, overlaps with Recommendation.Category
    Message      string    // redacted
    Detail       string    // redacted
    Step         string    // "render"|"validate"|"apply:configmap"|"apply:snd-validator"|...
    Completed    []string  // step IDs that succeeded before the failure
    Remaining    []string  // step IDs not attempted
    NextCommands []string  // recovery hints, e.g. "seictl bench down --slug <slug>"
}
```

Behavior:

- `--format=json`: full envelope to **stdout**, exit code set, nothing on stderr in the success-with-warnings case. On unexpected runtime errors (panic recovery, framework errors), fallback envelope with `category:"internal"` to stderr.
- `--format=text`: human-readable summary to stderr. Structured detail not shown unless `-v`. Exit code set.
- Partial apply: exit 15, `Completed` lists what's already in the cluster. Engineer is told to run `seictl bench down --slug <slug>` to clean up.

## Exit-code matrix

Locked: `0` success, `2` usage, `3` not-found, `4` cluster-unreachable, `5` permission-denied. Domain codes by family:

**Bench (10–19)**

| Code | Meaning |
|---|---|
| 10 | Image policy violation (registry/prefix mismatch, parse failure) |
| 11 | Image digest resolution failed (ECR not found / auth missing) |
| 12 | Input validation failed (slug, size, duration regex/range) |
| 13 | Namespace policy violation (`-n` mismatch with engineer alias) |
| 14 | Cluster headroom exceeded (>90%) |
| 15 | Partial apply failure (some resources created, some rejected) |
| 16 | Chain ID collision (slug already in use, alive bench) |
| 17 | Finalizer wait timed out (`bench down --wait`) |
| 18 | Template render failed |

**Onboard (20–29)**

| Code | Meaning |
|---|---|
| 20 | Alias invalid (regex or deny-list match) |
| 21 | Platform repo not found / not a git checkout |
| 22 | Git working tree dirty |
| 23 | `gh` not authenticated |
| 24 | PR creation failed (network / API) |
| 25 | Identity file present, alias mismatch on `--update` |

**Seinode/Controller (30–39)**

| Code | Meaning |
|---|---|
| 30 | SeiNode CRD not installed in cluster |
| 31 | Controller pod absent / no leader lease |
| 32 | Diagnose: data insufficient (events purged, no conditions) |

**Context/Identity (40–49)**

| Code | Meaning |
|---|---|
| 40 | Production-context refusal (matched prod pattern) |
| 41 | Identity file malformed |
| 42 | Identity file missing (when required by command) |
| 43 | Kubeconfig parse error |
| 44 | Identity file perms too loose (refuse-on-loose-perms) |

## Migration / drift / deprecation notes

- `seictl bench` JSON schemas (the result types above) are versioned via `Envelope.Version`. Bumping them is a breaking change for the `sei-platform-engineer` skill's reference content. Keep `v1` stable through MCP graduation.
- `Recommendation.Category` enum values: adding is non-breaking, renaming/removing is breaking. Treat the enum as a public API.
- Step IDs in `ErrorBody.Step` are public contract.
- Field manager `seictl-bench` is a one-way door — changing it abandons ownership of every existing object.
- Templates `UPSTREAM` SHA bumps that change rendered output should land via dedicated PRs with explicit notes; engineer-visible behavior should not silently change between seictl releases.
